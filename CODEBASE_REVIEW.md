# hpm Codebase Review

Generated by deepseek-v4-flash. Full analysis of the hpm repository.

## 🔴 Critical

### 1. CLI `query` hybrid mode is broken — never calls `query_hybrid`

**File:** `src/hpm/cli.py`, lines 78–84

When `mode="hybrid"`, the CLI runs **only vector search**, then falls back to keyword **only if vector returned nothing**. It never calls `db_module.query_hybrid()`. The `--mode hybrid` flag is effectively an alias for `--mode vector` with a silent keyword fallback on empty results.

**Fix:** Replace the two conditionals with a single call to `db_module.query_hybrid()` when `mode == "hybrid"`.

---

### 2. API key written in plaintext to `~/.hpm/.env`

**Files:** `src/hpm/config.py` (write_env), `src/hpm/cli.py` (setup command)

The `write_env()` function writes `KEY=VALUE` pairs to `~/.hpm/.env` in plaintext. No file permission restrictions are enforced (default umask).

**Fix:** At minimum, `os.chmod(env_file, 0o600)` after writing. Consider a keyring integration or macOS Keychain.

---

### 3. `HPM_ANSWER_MODEL` env var is dead config — never referenced

**File:** `src/hpm/config.py`, line 161; `src/hpm/llm.py`, line 98

`ANSWER_MODEL` is read from env in config.py but never used anywhere. In `llm.py` the model resolution chain is:
```python
resolved_model = model or config.SUMMARIZATION_MODEL or cfg["default_model"]
```
No reference to `ANSWER_MODEL`.

**Fix:** Wire it into `answer.py` or `llm.complete` resolution chain.

---

### 4. MCP server sends `initialized` notification before handshake

**File:** `hpm_mcp_server.py`, line 203–204

The JSON-RPC MCP protocol expects the server to first receive an `initialize` request from the client before sending `initialized`. Sending it unprompted before reading stdin is non-standard and may cause issues with some MCP clients.

**Fix:** Remove the proactive `send_notification("initialized", {})` call.

---

### 5. FTS5 query injection risk — no escaping of special characters

**File:** `src/hpm/db.py`, lines 462–472

FTS5 special characters like `*`, `(`, `)`, `NOT`, `OR`, `NEAR`, `^` inside user-provided terms can trigger unexpected FTS5 query behavior.

**Fix:** Strip/escape FTS5 control characters before quoting.

## 🟡 Should Fix

### 6. `query_keyword` returns raw FTS5 rank — confusing display
**File:** db.py:348-368, cli.py:98

FTS5's BM25 rank is negative (more negative = better match). The CLI displays it as-is. A perfect match shows as `score: -5.2345` which is non-obvious.

**Fix:** Normalize FTS5 rank to [0,1] in `query_keyword`.

### 7. Dead code in db.py
**File:** db.py:25-26

`_HERMES_MEMORIES_DIR` and the original `_DEFAULT_DB_PATH` are immediately shadowed by `config.HPM_DIR` at line 82. Should be cleaned up.

### 8. No `query_hybrid` edge case tests
Missing coverage for: fetch_size < limit, empty results, extreme vector_weight values (0.0 or 1.0), empty string query, all-identical vectors.

### 9. MCP server hard-depends on sentence-transformers
**File:** hpm_mcp_server.py:41

The MCP server imports `rerank` at module level. If `sentence-transformers` isn't installed (`pip install hpm[reranker]`), the MCP server crashes on startup.

**Fix:** Make the import lazy (inside `handle_memory_find`) or wrap in try/except.

### 10. Sidecar only checks `OPENGINE_API_KEY`, not active provider
**File:** sidecar.py:190-194

The warning is misleading when using Anthropic, OpenAI, or OpenRouter.

**Fix:** Check the key corresponding to `config.LLM_PROVIDER`.

### 11. Embedding model load failure has generic error
**File:** cli.py (generic except handlers)

If fastembed model download/load fails, the exception propagates to `except Exception: print(f"error: {exc}")`.

**Fix:** More specific error message about model loading.

### 12. Dashboard embeds all entry content into HTML
**File:** dashboard.py:66

If a memory entry accidentally captured an API key, it would be embedded in the dashboard HTML in plaintext.

**Fix:** Consider a content warning or sanitize-for-display option.

## 🟢 Nice to Have

### 13. MCP server doesn't handle `shutdown` notification

### 14. No tests for MCP server handlers
Zero tests for `hpm_mcp_server.py`.

### 15. No decay tests
`compute_decay_score`, `run_decay`, `reinforce` have zero test coverage. `decay.py`'s `run_spot_check` is also untested.

### 16. `superseded_by` column exists but dedup logic never sets it
The `merge_or_insert` function either merges or inserts new. In neither case does it set `superseded_by`. The column appears to be prepared for a conflict-resolution pattern that was never implemented.

### 17. No concurrency test for WAL mode + retry
The architecture specifies WAL mode with retry for concurrent access, but there are no tests exercising concurrent writers.

### 18. Missing `query_vector` vs `query_hybrid` schema difference
`query_hybrid` removes the `distance` key from results, while `query_vector` keeps it. Callers may not expect different schemas.
